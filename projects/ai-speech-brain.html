AI Speech Brain

Python Next.js + TypeScript Claude + GPT 2026 · Active

I give talks on the state of human data in AI. Keeping them current means tracking a constant stream of research talks, newsletters, and papers - far more than any one person can actually watch and read. So I built a system that ingests all of it and learns what I consider signal.

How it works

Every day a pipeline pulls whatever is new from my watchlist (YouTube channels, RSS newsletters, documents I drop in). One model extracts candidate "nuggets" - single, self-contained insights. A second model from a different vendor scores each nugget against my taste profile. Two vendors on purpose: I don't want to inherit any one lab's blind spots.

The day's top 8 show up as cards in a review app: approve, reject with a reason, edit, or skip. Each card comes with its own tools - I can ask it questions, request a simpler version, or pull more evidence from the source transcript before deciding.

The self-improvement loop

Reviews aren't just filtering - they're training signal. Every approve and reject (with its reason) gets added to a taste set. The judge reads that set on every run, so its ranking criteria get updated daily based on my latest decisions. The system also runs evals against my past decisions, which measures whether its judging accuracy is actually improving over time.

Stack choices

Python for the pipeline, Next.js 14 + TypeScript for the review app, Supabase for storage, Vercel for hosting. Claude does extraction and GPT does judging - splitting the two jobs across vendors is a design decision, not an accident of API keys. The daily run is split too: a scheduled cloud job handles decision syncing and feeds, and a local job on my Mac covers what doesn't work from datacenter IPs (YouTube transcripts, mostly).

A New SaaS Architecture Is Emerging

SaaS Agents

Our interaction with any SaaS product has always been determined by the existing UI and pre-built feature sets. However, any UI and any feature set has inherent tradeoffs. There are usually two strategies companies pursue when building the UI & feature sets:

1. Build for a core user persona and optimize everything around their needs

2. Create an interface that meets the needs of most users

The result is usually not only a limited user experience but also often complex feature stacks with steep learning curves. Entire job categories (e.g. Salesforce admins) exist due to such inherent software interface complexities.

All of that is about to change.

Models like Nano Banana are already capable of generating completely new UI interfaces with a click of the button. In the future users will no longer need to interact with standard UI. They’ll just create a view perfectly tailored to their needs and then will change that view as their needs change. Of course, some interfaces will likely stay static for user familiarity, compliance reasons etc. But there is no reason why going forward the bulk of our software interaction can't be 100% custom-tailored to our needs.

But the change is much deeper than UI.

Previously, users were constrained to whatever outputs the company's feature set enabled. Now, a new middle layer is emerging. The agentic layer. Going forward users will be customizing not only their interface but the actual output of the software. Agents can execute custom workflows and complete tasks specific to each user's request - generating new outputs customized for each user.

De facto we are transitioning from 2-layer SaaS architectures (Front-End + Back-End) to 3-layer architectures (Generative Front-End + Agent + Back-End).

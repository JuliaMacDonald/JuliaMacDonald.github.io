The 2026 Multimodal Training Data Roadmap

Voice Training Data

Voice is emerging as a key modality in 2026. It is on track to become the default user interface for a large share of agentic use cases from field worker support agents to nursing home agent companions. Here is what I think the multimodal training data roadmap will look like in 2026:

Voice-Enabled Agentic Task Completion Data

Labs will want training data that combines voice inputs with full agent trajectories. This data can range from "simple" cases: voice input + task completion ("Please book me a flight to Toronto and add it to my calendar") all the way to complex multi-turn speech-to-speech interactions with the agent asking questions along the way.

Voice-Enabled Agentic Task Completion Data with Intent / Prosody Captured

Multimodal data has many more “degrees of freedom” compared to text. A user saying “I am ok” to a companion agent in a nursing home can trigger completely different agent behavior depending on how these words are spoken. Training agents on such use cases will require data with prosody, intent and emotion labels.

Agents Handling 3+ Modalities

Consider a field worker looking at a machine they need to repair while wearing a headset and speaking to the agent: "Please tell me what type of break this is." The data to train such agents will need to align voice commands with visual context.

Multimodal RL Environments

Current RL environments function primarily as text-based digital twins of enterprise software (e.g Salesforce or Zendesk). I believe the next generation of RL environments will be audio-visual sandboxes where agents are required to process spoken or visual input while also interacting with a software interface. For example an agent could be trained to update patient records in an EHR system based on a doctor's dictated notes.

Embodied AI / Robotics (emerging)

"Pick up the box on the left" requires training data that maps spoken commands + visual input to physical action sequences. This in turn requires datasets that synchronize sensor readings, audio-visual input and resulting actions to ensure precise physical coordination.

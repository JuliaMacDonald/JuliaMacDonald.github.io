VLM Fine-Tuning Lab

PyTorch Hugging Face LoRA Apple Silicon 2026

After years of running AI data operations and advising teams on fine-tuning decisions, I wanted a current, hands-on reference point. So I fine-tuned a vision-language model end to end on my MacBook.

The setup

The task is visual question answering: show the model a photo, ask "what color is the umbrella?", get a grounded answer. I trained LoRA adapters on a slice of VQAv2 (the standard VQA benchmark - real photos with human-written questions and answers) using the Hugging Face Trainer on Apple's MPS backend. LoRA updates only a small fraction of the weights, which is what makes this feasible on a laptop.

The full workflow is in one notebook: hardware check, dataset inspection, training, and a before/after comparison of the model's answers.

Goals

Three goals: understand first-hand what a small fine-tune costs in time and compute, see where it fails and how, and keep a current reference point for the fine-tune vs better-data question that comes up constantly in my work with teams.

Stack choices

PyTorch with the MPS backend to run everything locally on Apple Silicon - no cloud GPU needed. Hugging Face Transformers and Datasets because they are the industry standard with the best documentation. PEFT for LoRA: the base model stays frozen and only small adapter matrices are trained, which keeps the memory footprint within laptop range. Everything is structured as a notebook so each step stays visible and reproducible - the data, the training run, and the model's answers before and after.

Synthetic data generation: you press a button, send your prompt to an LLM, get the data back and go train your model. Right?

Synthetic Data Pipelines

Let's take a look at a state-of-the-art synthetic pipeline based on Moonshot AI building agentic behavior in Kimi K2

Step 1: Tool spec generation

“We construct a tool repository. First, we directly fetch 3000+ real MCP tools from GitHub repositories. Second, we evolve [create] synthetic tools [..]: we begin with key categories (e.g., financial trading [..]), then we evolve specific application domains within each category. Specialized tools are then synthesized for each domain. This evolution process produces over 20,000 synthetic tools.”

Step 2: Agent profile generation

“We generate thousands of distinct agents by synthesizing various system prompts and equipping them with different combinations of tools from our repository.”

Step 3: Rubric-based task generation

“For each agent configuration, we generate tasks that range from simple to complex operations. Each task is paired with an explicit rubric that specifies success criteria, expected tool-use patterns, and evaluation checkpoints.“

Step 4: User simulation

Further, the Moonshot AI team creates “LLM-generated user personas with distinct communication styles and preferences.”

Step 5: A tool simulator executes multi-turn user <> agent interactions and tool calls + provides realistic feedback. Plus “we complement our simulated environments with real execution sandboxes for scenarios where authenticity is crucial, particularly in coding and software engineering tasks.“

Step 6: An LLM-Judge evaluates each trajectory against the task rubrics

Finally the model then is trained via supervised fine tuning and reinforcement learning.

Let's go over some of the human checkpoints in this process:

  • Decide on the categories and domains to cover
  • Curate the real MCP tools and verify licenses
  • Specify & generate inputs / outputs for synthetic tools
  • Prepare system prompt(s) for agents profile gen
  • Prepare system prompt(s) for simulated users profile gen
  • Prepare system prompt(s) for tasks and rubrics gen
  • Prepare system prompt for LLM-Judge
  • Configure the simulator + set up the execution sandboxes to capture full agent trajectories
  • Run data-gen in batches + review each batch for accuracy, safety etc
  • Remove near duplicates / balance the topic mix
  • Calibrate the LLM-Judge with human gold data sets
  • Approve quality trajectories as training data
  • Keep version history for prompts, tools etc

If this sounds like a crazy amount of work - that's because it is :)
